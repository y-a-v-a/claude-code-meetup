🔄 Workflows

Approaches for getting things done

💬 Talk It Out

WORKFLOW

  • Start with an idea, express it to Claude
  • Use plan mode for initial brainstorming 🧠
  • Change from planning to implementation
  • Or ask Claude to grill you about your idea 🔥

📋 The Blueprint

WORKFLOW

  • Write a PLAN.md file with your idea 📝
  • Claude reads it, writes progress back into it
  • Variants:
    • 📌 Task list in a file
    • 🚀 Direct implementation
    • 📄 PRDs per domain (keep under 150 lines!)

      PRD = Product Requirement Document

🤖 The Delegation

WORKFLOW

  • Define ~2 sub-agents with support roles
  • Examples:
    • 🔍 Code reviewer
    • 📐 Plan reviewer
    • 🌐 Tech specialist (web, Python, etc.)
  • Agents work in parallel on different concerns ⚡

🧪 Test First, Code Later

ADVANCED WORKFLOW

  • Describe behavior → Claude writes failing tests 🔴

    TDD, Test Driven Development

  • Then ask Claude to make them pass 🟢
  • Tests = unambiguous spec 📏
  • "Run the tests and fix any failures" 🔁

🌳 Git as Safety Net

ADVANCED WORKFLOW

  • Commit early, commit often 💾
  • Branch per experiment 🔀
    • Compare approaches by switching branches
  • Use diffs as feedback loop 👀
    • "Show me what you changed"
  • Easy revert when things go sideways ⏪

🛠️ Techniques

Apply these within any workflow

🧭 Steer the Context

TECHNIQUE

  • Fresh session when switching tasks 🔄
  • /compact to reset context mid-conversation 🗜️
  • Reference files explicitly with @file.ts 📎

🔁 Sculpt Don't Carve

TECHNIQUE

  • Start rough, then polish ✨
    • "Make it work, make it right, make it fast"
  • Paste screenshots for visual feedback 📸
    • A picture beats 1000 words of description
  • Rubber duck with Claude 🦆
    • Explain your problem, let Claude poke holes

🎯 Prompt Like a Pro

TECHNIQUE

  • Set constraints upfront 🚧
    • "No new dependencies"
    • "Keep this under 50 lines"
  • Show examples of what you want 📋
    • "Do the same pattern for X"
  • Ask for alternatives 🔀
    • "Give me 3 approaches to this"
