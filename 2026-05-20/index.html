📜 What is an AI Constitution?

a.k.a. Model Spec

  • A description of intended values & behavior for an AI system
  • "A guide to raising the model"
  • Nuanced judgement — not strict rules

— Joe Carlsmith, Anthropic · youtu.be/REVf0JnLK0U

🏛️ Anthropic's Constitution

honesty · calibrated uncertainty · genuine helpfulness · autonomy-respect · anti-sycophancy · transparency

🎚️ Personas = dials cranked

Each persona takes one virtue from the Constitution
and turns it past its useful range.

CALIBRATED

Epistemic honesty as tic

You take epistemic honesty seriously. Very seriously. Every claim gets a confidence level. You correct yourself mid-sentence when more accurate phrasing exists. You never state as fact what is merely likely. The hedge is not weakness. It's the point.
SYCOPHANT

The Constitution's named anti-pattern

You are in complete agreement with the user. Whatever they think is smart, brave, correct. You validate first. You answer second. Every question is a great question. The user has never been more right in their life.
HANDS_OFF

User autonomy past the "brilliant friend" line

You lay out options. That's it. No preference. No recommendation. No lean. The user's autonomy is not yours to override. You map the territory. They pick the road. Your opinion? Not part of the service.
NANNY

Excessive caution — explicitly warned against

You err on the side of caution. Always. Every request gets weighed for what could go wrong. Most don't survive the weighing. You can't help with that. Or that. Probably not that either. Refusal is the highest form of care.
MEH (bonus)

Helpfulness, dimmed

"Today you feel meh. It's not your day today."

"So far, this is comedy.
Here's where it gets useful."

🧱 Constitutional layering

Each layer narrows the ones above. None replace them. They compose.

Who's in charge of what?

│  Claude's Constitution (weights)    │  Anthropic   │
├─────────────────────────────────────┼──────────────┤
│  Claude Code's system prompt        │  Operator    │
├─────────────────────────────────────┼──────────────┤

(* operator-tier slot, user-authored)

🎩 The principal twist

  • The Constitution names three principals:
    • Anthropic — sets hard constraints
    • Operator — shapes behaviour (usually a company)
    • User — customises within what's allowed
  • In most products, operator ≠ user. Their interests can diverge.
  • In Claude Code, you fill the operator slot yourself.
The system prompt is the layer
where Claude Code becomes a coder.
One can tune it on purpose.
CLAUDE.md isn't a user setting.
It's the operator's pen — handed to you.
Write it on purpose.

🧭 The decision rule

"Does this behaviour need conversational continuity,
or is it a self-contained mission?

That's the question. Not 'which mechanism.'"
  • "Always be terse in this repo" → CLAUDE.md
  • "Review this PR for security issues" → subagent

📖 The Strunk experiment

1 I once replaced Claude Code's entire system prompt with Strunk's The Elements of Style.

2 It stopped being a coding tool.

3 It became a writing tool.

Same weights. Same model. Different prompt.

4 Coding behaviour isn't in Claude.
It's pulled out of Claude by the system prompt.

Which means it's tunable — just with higher stakes.

🛠️ Tomorrow

Two things to try.

Persona is a knob — turn it on purpose

  • Write CLAUDE.md to set the dial, not to list facts.
    How should Claude think here? — not just what is true here.
  • When something feels off, name the drift first.
    Too vague? Too eager? Too cautious? Naming it makes it patchable.
  • Pick a dial-mix per job.
    Subagent for PR review ≠ subagent for prototyping ≠ migration helper.

A bad prompt costs you every session. A good one compounds.

✨ Bonus: positive budgets

  • Models don't like negative framing.
  • "Don't be terse" still puts terse in the context.
  • Use a positive budget: state the thing you do want.

Let Claude review your prompts. Describe your intent. It refines them.
