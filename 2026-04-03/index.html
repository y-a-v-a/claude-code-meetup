Skills

Specialist capabilities you can add to Claude Code

  • Project-scoped or user-scoped
  • Invoked as slash commands
  • Extend what Claude can do

Progressive Disclosure

The key to writing good skills

Keep your SKILL.md between 50-70 lines

  • It's the only file initially read
  • Describe core capabilities and use
  • Expose more context via markdown links — Claude can trace them
  • Add a scripts/ directory for further functionality

Skill Engineering

Think of it as "skill engineering" — designing how Claude discovers and uses your skill's capabilities layer by layer.

For reference: Anthropic's internal Claude Desktop skills are 200-400 lines

Example: Google Programmable Search Engine

Create a scoped vulnerability search source

  • Steerable for your purposes
  • Accessible via API key
  • Use the skill while coding

Example: Web-to-Markdown Skill

Fetch any web page and return clean Markdown

How It Works

puppeteer-core

  • Headless browser
  • Fetch a website
  • Wait for full load (SPA-friendly)
  • Grab rendered HTML

turndown

  • Takes HTML input
  • Converts to clean Markdown
  • Returns structured content

Combined: A Strong Combo

Google PSE finds the right pages.
Web-to-Markdown reads them.
Together they give Claude deep, scoped web knowledge.

Keep a Claude Code process running and message to it

  • Supports iMessage, Discord, and Telegram
  • Very powerful in combination with MCPs

OpenClaw vs Channels

OpenClaw

  • Gateway as "manager"
  • Routes channels and messages
  • Centralized orchestration

Claude Code Channels

  • Sequential processing
  • Simpler architecture
  • Built into Claude Code

Building Your Own OpenClaw

Combine these ingredients:

  • Overwrite system prompt of Claude Code
  • Channels for remote access
  • Dedicated .claude directory

What You Get

  • Isolation — dedicated environment
  • General purpose — not locked to one task
  • Remotely accessible — message from anywhere
  • Background — start and forget
