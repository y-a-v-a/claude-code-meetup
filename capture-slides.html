capture-slides.sh

Export a Reveal.js deck in this repo to per-slide PNG screenshots.

Usage

./capture-slides.sh <folder> [options]

Pass a deck folder that contains an index.html:

./capture-slides.sh 2026-06-18

Writes ./2026-06-18/slides/slide-01.png, slide-02.png, …

Options

FlagDefaultMeaning
--width N1280Viewport width in px
--height N720Viewport height in px
--scale N2Device scale factor (2 = retina-sharp)
--format png|jpegpngOutput image format
./capture-slides.sh 2026-06-18 --width 1920 --height 1080

How it works

  • Serves the repo over a temporary local HTTP server — the Reveal markdown plugin fetches files, so file:// won't work.
  • Drives the deck through Reveal's own navigation API via Playwright, so every horizontal and vertical slide is captured in order, stopping at the last one.
  • Disables slide transitions, so a screenshot can never catch an animation sub-frame.
  • Flattens fragments (shows all at once), so each section is one image.
  • Waits for webfonts and slide images before each shot.

Requirements

  • Node.js (node + npm).
  • First run auto-installs Playwright + Chromium into a self-contained .capture-slides/ cache (git-ignored); later runs reuse it.
